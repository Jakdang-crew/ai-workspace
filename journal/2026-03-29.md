## 배운 것: Git Submodule

### 왜 사용하는가?
- 메인 워크스페이스와 각 프로젝트가 독립적인 git history를 가짐
- 전체 워크스페이스 상태를 특정 시점으로 복원 가능

### 새 프로젝트 추가 방법

**방법 1: GitHub에서 시작 (권장)**
```bash
# GitHub에서 저장소 생성 후
git submodule add git@github.com:user/project.git projects/project-name
```

**방법 2: 로컬에서 시작**
```bash
# 1. 프로젝트 생성 및 초기화
cd projects && mkdir my-project && cd my-project && git init

# 2. 작업 후 GitHub에 빈 저장소 생성 (README 없이)

# 3. 원격 연결 및 푸시
git remote add origin git@github.com:user/my-project.git
git push -u origin main

# 4. submodule 등록 (폴더 이동 방식 - 간단)
cd ~/ai-workspace
mv projects/my-project ../backup
git submodule add git@github.com:user/my-project.git projects/my-project
rm -rf ../backup
```

### 일상적인 작업 흐름
```bash
# 프로젝트에서 작업 후
cd projects/my-project
git add . && git commit -m "작업 내용" && git push

# 메인 워크스페이스에서 참조 업데이트
cd ~/ai-workspace
git add projects/my-project
git commit -m "Update my-project submodule"
git push
```

### 클론할 때
```bash
git clone --recursive <repo-url>
# 또는
git clone <repo-url> && git submodule update --init --recursive
```

## 느낀 점

AI와 함께 성장하기 위한 첫 발걸음을 내딛었다. 기록을 꾸준히 남기는 것이 중요하다. Git submodule은 처음엔 복잡해 보이지만, 프로젝트별 독립 관리가 필요할 때 유용하다.
